## 기능: 플로팅 바에 앱 추가
 5 시나리오 
 
 
 배경: FloatMe가 실행 중이고 플로팅 바가 화면에 표시되어 있다. 
 
 
시나리오 1: + 버튼으로 실행 중인 앱을 플로팅에 추가
  
- 만일 사용자가 플로팅 바의 + 버튼을 클릭하면
 
- 그러면 현재 실행 중인 앱 목록이 팝업으로 표시된다
 
- 그리고 이미 플로팅에 등록된 앱은 체크마크로 구분된다
 
- 그리고 미등록 앱을 클릭하면 즉시 플로팅 바에 아이콘이 추가된다
 
 
 
 
시나리오 2: 검색으로 앱 필터링하여 추가
  
- 만일 앱 선택 팝업이 열린 상태에서
 
- 그리고 사용자가 검색 필드에 "Xco"를 입력하면
 
- 그러면 "Xcode"가 포함된 앱만 필터링되어 표시된다
 
 
 
 
시나리오 3: 이미 추가된 앱을 다시 추가 시도 (엣지 케이스)
  
- 만일 Xcode가 이미 플로팅에 등록된 상태에서
 
- 그리고 사용자가 팝업에서 Xcode를 클릭하면
 
- 그러면 Xcode가 플로팅에서 제거된다 (토글 동작)
 
 
 
 
시나리오 4: 최대 개수 초과 시 경고
  
- 만일 이미 15개 앱이 플로팅에 등록된 상태에서
 
- 그리고 사용자가 추가 앱을 등록하려 하면
 
- 그러면 "플로팅 바가 너무 길어질 수 있습니다" 경고가 표시된다
 
- 하지만 사용자가 확인하면 추가가 허용된다
 
 
 
 
시나리오 5: 실행 중인 앱이 없을 때 (빈 상태)
  
- 만일 Finder 외에 실행 중인 앱이 없는 상태에서
 
- 그리고 + 버튼을 클릭하면
 
- 그러면 Finder만 표시되는 팝업이 나타난다
 
 
 
 
  
 
 

## 기능: 앱 전환 (클릭)
 4 시나리오 
 
 
 배경: 플로팅 바에 Xcode, Safari, Slack이 등록되어 있고 현재 Safari가 활성 상태이다. 
 
 
시나리오 1: 정상 앱 전환
  
- 만일 사용자가 Xcode 아이콘을 클릭하면
 
- 그러면 Xcode의 모든 윈도우가 최상단으로 올라온다
 
- 그리고 Xcode 아이콘에 활성 하이라이트가 표시된다
 
- 그리고 Safari 아이콘의 활성 하이라이트가 해제된다
 
 
 
 
시나리오 2: 이미 활성인 앱을 클릭
  
- 만일 Safari가 이미 활성 상태에서 Safari 아이콘을 클릭하면
 
- 그러면 Safari의 모든 윈도우가 다시 최상단으로 정렬된다
 
 
 
 
시나리오 3: 종료된 앱 아이콘 클릭 (에러)
  
- 만일 Xcode가 종료되었는데 플로팅에 아직 잔존하는 상태에서
 
- 그리고 사용자가 Xcode 아이콘을 클릭하면
 
- 그러면 Xcode를 실행하려고 시도한다 (NSWorkspace.open)
 
 
 
 
시나리오 4: 연속 빠른 클릭 (더블 클릭 방지)
  
- 만일 사용자가 Xcode 아이콘을 200ms 이내에 두 번 클릭하면
 
- 그러면 앱 전환은 한 번만 수행된다 (디바운스)
 
 
 
 
  
 
 

## 기능: 플로팅에서 앱 제거
 3 시나리오 
 
 
 
시나리오 1: 우클릭 메뉴로 제거
  
- 만일 사용자가 앱 아이콘을 우클릭하면
 
- 그러면 컨텍스트 메뉴가 표시된다
 
- 그리고 "플로팅에서 제거"를 클릭하면 아이콘이 즉시 사라진다
 
- 그리고 나머지 아이콘이 빈 자리를 채우도록 재정렬된다
 
 
 
 
시나리오 2: Option+클릭으로 빠른 제거
  
- 만일 사용자가 Option 키를 누른 채 앱 아이콘을 클릭하면
 
- 그러면 확인 없이 즉시 플로팅에서 제거된다
 
 
 
 
시나리오 3: 마지막 앱 제거 시 빈 상태
  
- 만일 플로팅에 앱이 1개만 남은 상태에서 제거하면
 
- 그러면 + 버튼만 남은 최소 크기의 플로팅 바가 표시된다
 
 
 
 
  
 
 

## 기능: 플로팅 바 이동
 4 시나리오 
 
 
 
시나리오 1: Cmd+드래그로 이동
  
- 만일 사용자가 Cmd 키를 누른 상태에서 아이콘을 드래그하면
 
- 그러면 플로팅 바 전체가 마우스를 따라 이동한다
 
- 그리고 드래그 종료 시 새 위치가 UserDefaults에 저장된다
 
 
 
 
시나리오 2: Cmd 없이 클릭은 앱 전환
  
- 만일 사용자가 Cmd 키 없이 아이콘을 클릭하면
 
- 그러면 이동이 아닌 앱 전환이 수행된다
 
 
 
 
시나리오 3: 화면 밖으로 이동 시도 (경계 처리)
  
- 만일 사용자가 플로팅 바를 화면 경계 밖으로 드래그하면
 
- 그러면 플로팅 바가 화면 경계에 걸려 멈춘다 (최소 절반은 화면 안)
 
 
 
 
시나리오 4: 드래그 중 Cmd 키 해제
  
- 만일 드래그 중에 Cmd 키를 먼저 놓으면
 
- 그러면 드래그가 즉시 종료되고 현재 위치에 고정된다
 
 
 
 
  
 
 

## 기능: 앱 실행/종료 감지
 4 시나리오 
 
 
 
시나리오 1: 등록된 앱이 종료될 때 (자동 제거 ON)
  
- 만일 "종료된 앱 자동 제거" 설정이 켜져 있고
 
- 그리고 플로팅에 등록된 Xcode가 종료되면
 
- 그러면 Xcode 아이콘이 플로팅 바에서 자동 제거된다
 
 
 
 
시나리오 2: 등록된 앱이 종료될 때 (자동 제거 OFF)
  
- 만일 "종료된 앱 자동 제거" 설정이 꺼져 있고
 
- 그리고 플로팅에 등록된 Xcode가 종료되면
 
- 그러면 Xcode 아이콘이 50% 투명도로 흐리게 표시된다
 
- 그리고 클릭 시 Xcode 재실행을 시도한다
 
 
 
 
시나리오 3: 앱 재시작 후 설정 복원
  
- 만일 FloatMe를 재시작하면
 
- 그러면 이전에 저장된 앱 목록, 위치, 설정이 복원된다
 
- 그리고 현재 실행 중인 앱만 활성 상태로 표시된다
 
 
 
 
시나리오 4: 다른 앱에서 앱 전환 시 활성 표시 갱신
  
- 만일 사용자가 Cmd+Tab이나 Dock으로 앱을 전환하면
 
- 그러면 플로팅 바의 활성 하이라이트가 새로 포커스된 앱으로 즉시 이동한다
 
 
 
 
  
 
 

## 기능: 설정 관리
 4 시나리오 
 
 
 
시나리오 1: 레이아웃 방향 전환
  
- 만일 메뉴바에서 "방향" 항목을 클릭하면
 
- 그러면 가로 ↔ 세로가 즉시 전환된다
 
- 그리고 전환 애니메이션이 부드럽게 적용된다
 
 
 
 
시나리오 2: 아이콘 크기 조절
  
- 만일 환경설정에서 아이콘 크기 슬라이더를 조절하면
 
- 그러면 플로팅 바의 아이콘 크기가 실시간으로 변경된다
 
- 그리고 24px ~ 48px 범위 내에서 조절 가능하다
 
 
 
 
시나리오 3: 플로팅 바 표시/숨김 토글
  
- 만일 메뉴바에서 "플로팅 바 표시"를 토글하면
 
- 그러면 플로팅 바가 표시되거나 숨겨진다
 
- 그리고 Cmd+Shift+F 단축키로도 동일하게 토글 가능하다
 
 
 
 
시나리오 4: 전체화면 진입 시 자동 숨김
  
- 만일 "전체화면에서 숨기기" 설정이 켜져 있고
 
- 그리고 사용자가 앱을 전체화면으로 전환하면
 
- 그러면 플로팅 바가 자동으로 숨겨진다
 
- 그리고 전체화면을 나가면 다시 표시된다
