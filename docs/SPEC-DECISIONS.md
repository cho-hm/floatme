## 3. 플로팅 바 동작

### 레이아웃 방향
- **가로 + 세로 모두 지원** (설정에서 전환)
- 기본값: 가로

### 오버플로우 처리: 자동 크기 축소
- 아이콘 수가 늘어나면 아이콘 크기를 자동으로 줄여서 화면 안에 맞춤
- macOS Dock과 유사한 방식
- **최소 크기 제한: 20px** (이하로는 축소 불가)
- 최소 크기에 도달 후에도 추가하면 화면 경계를 넘어갈 수 있음

### 스냅 기능: 화면 가장자리 스냅
- 화면 끝에서 20px 이내에 오면 자동으로 가장자리에 달라붙음
- Shift 키를 누르면 스냅 무시 (자유 배치)

### 다중 모니터
- **하나의 플로팅 바**, 모니터 간 자유 이동
- Cmd+드래그로 모니터 경계를 넘어 이동 가능
- NSScreen 감지로 경계 스냅 적용

### 바 이동: Cmd+드래그
- Cmd 키를 누른 상태에서 드래그하면 플로팅 바 전체 이동
- 편집 모드와 별개로 독립 동작
- 드래그 종료 시 위치 UserDefaults에 저장

## 4. 편집 모드

### 진입: 길게 누르기 (Long Press)
- 플로팅 바의 아이콘을 0.5초 이상 길게 누르면 편집 모드 진입
- iOS 홈화면 편집과 유사한 UX

### 편집 모드 기능
1. **아이콘 흔들림 애니메이션** — 편집 모드임을 시각적으로 전달
2. **X 제거 버튼** — 각 아이콘 좌상단에 X 버튼 표시, 클릭 시 즉시 제거
3. **+ 추가 버튼** — 바 끝에 + 버튼 나타남, 클릭 시 앱 선택 팔레트 열림
4. **드래그 정렬** — 아이콘을 드래그하여 순서 변경

### 편집 모드 종료
- 플로팅 바 바깥 영역 클릭 시 종료
- ESC 키 누를 시 종료

```
일반 모드:
┌───────────────────────┐
│ [Xc] [Sa] [Fi] [Sl]  │   ← 클릭 = 앱 전환
└───────────────────────┘   ← Cmd+드래그 = 바 이동

편집 모드 (길게 누른 후):
┌──────────────────────────────┐
│  ×     ×     ×     ×         │
│ [Xc~] [Sa~] [Fi~] [Sl~] [+] │   ← 드래그 = 순서 변경
│                              │   ← × = 제거
└──────────────────────────────┘   ← [+] = 앱 추가
```

## 5. 앱 추가/제거

### 앱 추가 방법 (3가지 모두 지원)
1. **편집 모드의 + 버튼** → 앱 선택 팔레트 표시
2. **메뉴바 메뉴** → "앱 추가" 항목에서 동일한 팔레트 호출
3. **글로벌 핫키** → 키보드 단축키(예: Cmd+Shift+A)로 팔레트 즉시 열기

### 앱 제거 방법
1. **편집 모드의 X 버튼** (주요 방법)
2. **우클릭 컨텍스트 메뉴** → "플로팅에서 제거"
3. **Option+클릭** → 즉시 제거 (빠른 제거)
4. **앱 선택 팔레트에서 토글** → 이미 추가된 앱 클릭 시 제거

### 앱 종료 시 처리: 자동 제거
- 플로팅에 등록된 앱이 종료되면 즉시 자동 제거
- "지금 쓸 수 있는 앱"만 표시하는 동적 세트 철학

### 앱 필터링
- `activationPolicy == .regular`인 앱만 표시 (Dock에 아이콘이 보이는 일반 앱)
- FloatMe 자신은 목록에서 숨김
- Finder는 표시 (윈도우를 올리는 의미가 있으므로)

## 6. 시각적 디자인

### 배경 스타일: 사용자 선택
- **기본값**: macOS 네이티브 블러 (NSVisualEffectView)
- **옵션**: 블러(시스템) / 다크(불투명) / 투명
- 환경설정에서 전환 가능
- 투명도 슬라이더 (50% ~ 100%)

### 현재 활성 앱 표시: 도트 + 테두리 병행
- **하단 도트**: 실행 중인 앱에 작은 파란 점 (Dock 스타일) — 자동 제거 모드에서는 모든 앱이 실행 중이므로 사실상 불필요하나, 향후 유령 모드 지원 시 의미 있음
- **테두리 하이라이트**: 현재 포커스된(최전면) 앱에 파란 테두리(2px ring)
- 두 상태를 동시에 전달

```
  [Xc]  ┌[Sa]┐  [Fi]  [Sl]
   ●    └ ● ─┘   ●
  run  focused   run
```

### 호버 피드백: 툴팁만
- 마우스 호버 시 앱 이름 툴팁 표시
- 아이콘 확대/마그니피케이션 없음

### 추가/제거 애니메이션: 부드러운 애니메이션
- 추가 시: 페이드인 + 슬라이드
- 제거 시: 축소 + 페이드아웃
- SwiftUI `withAnimation`으로 구현

## 7. 인터랙션 모델 (확정)

| 입력 | 대상 | 동작 |
|------|------|------|
| **클릭** | 플로팅 아이콘 | 해당 앱의 **모든 윈도우** 최상단 활성화 |
| **우클릭** | 플로팅 아이콘 | 컨텍스트 메뉴 (제거, 앱 정보) |
| **Option+클릭** | 플로팅 아이콘 | 즉시 제거 |
| **길게 누르기 (0.5초)** | 플로팅 바 | 편집 모드 진입 (흔들림 + X + + 버튼) |
| **Cmd+드래그** | 플로팅 바 | 플로팅 바 전체 이동 |
| **편집 모드 드래그** | 아이콘 | 아이콘 순서 변경 |
| **편집 모드 X 클릭** | 아이콘 X 버튼 | 아이콘 제거 |
| **편집 모드 + 클릭** | + 버튼 | 앱 선택 팔레트 열기 |
| **메뉴바 클릭** | 메뉴바 아이콘 | 설정/앱 추가/표시 토글 |
| **Cmd+Shift+A** | 글로벌 | 앱 선택 팔레트 열기 |
| **Cmd+Shift+F** | 글로벌 | 플로팅 바 표시/숨김 토글 |

## 8. 앱 생명주기

### 앱 감지: NSWorkspace Notification
- `didLaunchApplicationNotification` — 앱 실행 감지
- `didTerminateApplicationNotification` — 앱 종료 감지 → 플로팅에서 자동 제거
- `didActivateApplicationNotification` — 활성 앱 추적 → 테두리 하이라이트 갱신

### 초기 로드
- `NSWorkspace.shared.runningApplications`에서 `.regular` 앱 목록 로드
- UserDefaults에서 이전 세션의 고정 앱 목록(Bundle ID) 복원
- 복원된 앱 중 현재 실행 중인 앱만 플로팅에 표시

### 멀티 윈도우 처리
- 클릭 시 `activate(options: .activateAllWindows)` — 해당 앱의 모든 윈도우 최상단
